# Security Policy

## Supported Versions

During pre-launch, only the latest commit on the `main` branch is supported. Once we tag releases, this section will be updated to reflect supported version ranges.

## Reporting a Vulnerability

If you discover a security vulnerability in this repository or in pinka.finance services, **please do not open a public GitHub issue**.

Instead, email **[security@pinka.finance](mailto:security@pinka.finance)** with:

- A clear description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Suggested fix (if any)
- Whether you wish to be acknowledged publicly after disclosure

If you prefer encrypted communication, request our PGP key in your initial message.

## Our Commitments

- **Acknowledgment** of your report within 48 hours
- **Initial assessment** within 7 days
- **Coordinated disclosure** — we won't publish details until a fix is available, and we'll work with you on disclosure timing
- **Public credit** (with your permission) once the issue is resolved

## Scope

This repository contains the public landing page only. For vulnerabilities in:

- Smart contracts → see the contracts repository (link added on publication)
- Platform infrastructure → email the security address above
- Monerium platform → report directly to Monerium ([https://monerium.com](https://monerium.com))
- Safe contracts → report directly to Safe ([https://safe.global](https://safe.global))

## Out of Scope

- Issues that require physical access to a victim's device
- Social engineering of pinka staff or users
- Attacks requiring privileged credentials obtained outside this repo
- Vulnerabilities in third-party services we depend on (please report those upstream)

## Acknowledgments

We're grateful to security researchers who report responsibly. A public acknowledgments section will be maintained here once we have our first reports resolved.
